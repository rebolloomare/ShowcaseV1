Widget de Carga de Archivo


Forma de Uso

Agregando .fileinput al contenedor.
Los elementos dentro del contenedor con .fileinput-new y .fileinput-exists se muestra u ocultan basados en su estado actual. Una vista previa del archivo seleccionado se coloca en .fileinput-preview. El texto del .fileinput-filename se establece de acuerdo al archivo seleccionado.


El widget File Input debe ser colocado en un <form> reemplazando a un <input type="file"> convencional. La carga del archivo se maneja de manera normal del lado del servidor.


Atributos

Agregando data-provides="fileinput" al elemento .fileinput.
Implementar un botón para borrar el archivo con data-dismiss="fileinput". Agregar data-trigger="fileinput" a cualquier elemento dentro del widget .fileinput para activar el cuadro de diálogo del archivo.


Invocar mediante JavaScript

$('.fileinput').fileinput()

Forma

Usando los elementos dados, puede formar el widget Upload de la manera que desee, ya sea con dimensiones fijas con width y height o relativas con max-width y max-height.


Opciones

Name type description
name string Use esta opción en lugar del atributo name en el elemento <input> para evitar que sea parte de los datos de envío.

Métodos

.fileinput(options)

Inicializa el widget file upload.

.fileinput('clear')

Borra el archivo seleccionado.

.fileinput('reset')

Restablece el elemento del formulario al valor inicial.


Eventos

Evento Descripción
change.bs.fileinput Este evento se activa después de seleccionar un archivo.
clear.bs.fileinput Este evento se activa cuando se borra el archivo ingresado.
reset.bs.fileinput Este evento se activa cuando se restaura el archivo ingresado.
