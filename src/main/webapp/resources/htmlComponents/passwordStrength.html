Password Strength Plugin

jQuery Password Strength Meter es un plugin para Twitter Bootstrap que proporciona conjuntos de reglas para mostrar visualmente la calidad de una contraseña escrita por un usuario.

Uso

  1. Descargar e incluir las librerias JavaScript de jQuery Password Strength Meter en la sección de cabecera de su página. 
    <script type="text/javascript" src="<c:url value='/resources/jquery/plugins/passwordStrength/pwstrength-bootstrap.js'/>"></script>
  2. A continuación, sólo invocar el plugin en los campos de contraseña que desee adjuntar un medidor de seguridad. 
    $(':password').pwstrength();
  3. Para aplicarlo únicamente a una entrada y cambiar las opciones : 
    $('#passwd1').pwstrength({
        ui: { showVerdictsInsideProgressBar: true }
    });


Opciones

El plugin espera las opciones con la siguiente estructura :

Comunes

  • minChar:

    Default: 6 (Integer)

    Establece el mínimo de caracteres requerido para que una contraseña no sea considerada muy debil.

  • usernameField:

    Default: "#username" (String)

    El campo de nombre de usuario para que coincida con una contraseña para , para garantizar que el usuario no utiliza el mismo valor para su contraseña.

  • userInputs:

    Default: [] (Array)

    Gama de selectores CSS para los campos de entrada con la entrada del usuario. El contenido de estos campos serán recuperados y pasados a la biblioteca zxcvbn

    Esta opción sólo tiene efecto cuando se está utilizando la biblioteca zxcvbn . Comprobar la opciónzxcvbn.

  • onLoad:

    Default: undefined (Function)

    función de devolución de llamada, disparada en la carga del widget. No se pasaran argumentos.

  • onKeyUp:

    Default: undefined (Function)

    función de devolución de llamada, disparada en la tecla hacia ariba cuando el usuario está escribiendo. El keyup evento se pasa como primer argumento, y un objeto como el segundo lugar con la puntuación y el texto de veredicto y nivel

    Este controlador también se llamará cuando los eventos change o onpaste ocurran.

  • zxcvbn:

    Default: false (Boolean)

    Utilice zxcvbn para calcular la entropía de la contraseña y poder utilizarla como la puntuación.

    Si se activa esta opción, entonces todas las reglas no se aplican, y todas las opciones bajo la sección Rules se deberán ignorar también. zxcvbn será utilizado en lugar del motor de reglas por defecto.

    Para utilizar esta opción, se debe cargar el archivo de la biblioteca zxcvbn en su sitio.

  • zxcvbnTerms:

    Default: [] (Array)

    Conjunto de cadenas. Una lista de palabras prohibidas para la biblioteca zxcvbn. Esta opción será ignorada si no se está utilizando zxcvbn.

  • events: ["keyup", "change", "paste"] (Array)

    Conjunto de cadenas. Una lista de los nombres que los eventos del plugin va a escuchar. Personalizar esta opción para hacer frente al error en iOS 9 keyup.

  • debug:

    Default: false (Boolean)

    Si tiene el valor true, se imprime la seguridad de la contraseña en la consola de Javascript , por lo que puede probar y ajustar la configuración de reglas.

Reglas

  • extra:

    Default: {} (Object)

    objeto vacío donde se almacenarán las funciones de reglas personalizadas .

  • scores:

    Default: (Object)

    {
  wordNotEmail: -100,
  wordLength: -50,
  wordSimilarToUsername: -100,
  wordSequences: -50,
  wordTwoCharacterClasses: 2,
  wordRepetitions: -25,
  wordLowercase: 1,
  wordUppercase: 3,
  wordOneNumber: 3,
  wordThreeNumbers: 5,
  wordOneSpecialChar: 3,
  wordTwoSpecialChar: 5,
  wordUpperLowerCombo: 2,
  wordLetterNumberCombo: 2,
  wordLetterNumberCharCombo: 2
}

    Las puntuaciones devueltas por las reglas cuando coincidan. Los valores negativos corresponden a penalizaciones .

  • activated:

    Default: (Object)

    {
  wordNotEmail: true,
  wordLength: true,
  wordSimilarToUsername: true,
  wordSequences: true,
  wordTwoCharacterClasses: false,
  wordRepetitions: false,
  wordLowercase: true,
  wordUppercase: true,
  wordOneNumber: true,
  wordThreeNumbers: true,
  wordOneSpecialChar: true,
  wordTwoSpecialChar: true,
  wordUpperLowerCombo: true,
  wordLetterNumberCombo: true,
  wordLetterNumberCharCombo: true
}

    Un objeto que establece que reglas de validación se activan. Al cambiar este objeto es posible desactivar algunas validaciones, o activarlos para mayor seguridad.

  • raisePower:

    Default: 1.4 (Double)

    El valor que se utiliza para modificar la puntuación final, en base a la longitud de la contraseña, le permite adaptar sus resultados .

Interfaz de Usuario (UI)

  • bootstrap2:

    Default: false (Boolean)

    Establece si es compatible con el legado Bootstrap 2 (true) o el actual Bootstrap 3 (False), el HTML de la barra de progreso es diferente

  • bootstrap4:

    Default: false (Boolean)

    Establece si es compatible con Bootstrap inestable 4 (true) o el actual Bootstrap 3 (False), el HTML de la barra de progreso es diferente. Tenga en cuenta que la actual Boostrap 4 tiene soporte muy básico

  • colorClasses:

    Default: ["danger", "danger", "danger", "warning", "warning", "success"] (Array)

    Las clases CSS aplicados a la barra en la barra de progreso y de los veredictos , dependiendo de la fuerza de la contraseña.

    Tenga en cuenta que para boostrap 2 se añadirá el prefijo bar- , que para Boostrap 3 el prefijo será progress-bar , y que para Bootstrap 4 será progress- . Este es el caso de la barra de progreso, no de los veredictos. Para los veredictos no hay prefijo en absoluto

  • showProgressBar:

    Default: true (Boolean)

    Muestra la fuerza de la contraseña en una barra de progreso

  • progressBarEmptyPercentage:

    Default: 1 (Integer)

    Porcentaje mínimo lleno en la barra de progreso que representa la fuerza de la contraseña. Una contraseña vacía se mostrará la barra de progreso llena solo un tanto.

  • progressBarMinPercentage:

    Default: 1 (Integer)

    Porcentaje mínimo lleno en la barra de progreso que representa la fuerza de la contraseña. Una contraseña no muy buena pero no vacía mostrará la barra de progreso llena un tanto.

  • progressBarExtraCssClasses:

    Default: "" (String)

    Clases CSS que se añaden a la barra de progreso generada. Tiene el propósito de hacer uso de las clases adicionales proporcionadas por Bootstrap. Las clases serán añadidas a el elemento DOM adecuado dependiendo de qué versión de Bootstrap se está usando.

  • showPopover:

    Default: false (Boolean)

    Muestra los mensajes de error y los veredictos en un popover Bootstrap, en vez de debajo del campo de entrada. se deben incluir tooltip.js y popover.js.

    Si la opción showVerdictsInsideProgressBar esta activa, entonces los veredictos no aparecerán en el popover.

  • popoverPlacement:

    Default: "bottom" (String)

    Elige donde el popover debe ser colocado en relación con el campo de entrada. Las opciones posibles son: "top" "bottom" "left" "right" "auto"

  • showStatus:

    Default: false (Boolean)

    Muestra la fuerza de la contraseña como un estado de validación en el campo de la contraseña, para que esto funcione, se debe seguir la estructura de la forma de Bootstrap.

  • spanError:

    Default: (Function)

    function (options, key) {
  var text = options.i18n.t(key);
  if (!text) { return ''; }
  return '<span style="color: #d52929">' + text + '</span>';
};

    Función para generar un span con la propiedad de estilo ajustado a color de fuente rojo, utilizado en los mensajes de errores. Sobrescribir para un estilo personalizado.

  • popoverError:

    Default: (Function)

    function (errors) {
  var errors = options.instances.errors,
      errorsTitle = options.i18n.t("errorList"),
      message = "<div>" + errorsTitle + "<ul class='error-list' style='margin-bottom: 0;'>";

  jQuery.each(errors, function (idx, err) {
    message += "<li>" + err + "</li>";
  });
  message += "</ul></div>";
  return message;
};

    Función para generar la lista de errores en el popover si showPopover tiene el valor de true. Sobrescribir para un estilo personalizado.

  • showVerdicts:

    Default: true (Boolean)

    Determina si los veredictos se muestran o no

  • showVerdictsInsideProgressBar:

    Default: false (Boolean)

    Determina si los veredictos se muestran dentro de la barra de progreso o no. Cuando este valor está activo, la ventana gráfica del veredicto se ignora y no aparecerá en el popover si está esta siendo mostrada. También esta opción prevalece sobre el valor de los showVerdicts.

    Esta opción no está disponible cuando se utiliza Bootstrap 4.

  • useVerdictCssClass:

    Default: false (Boolean)

    Determina si es necesario añadir una clase css en el elemento veredicto.

  • showErrors:

    Default: false (Boolean)

    Determina si se muestra la lista de errores con la barra de progreso o no.

  • showScore:

    Default: false (Boolean)

    Determina si la puntuación de contraseña se muestra con la barra de progreso o no.

  • container:

    Default: undefined (CSS selector, or DOM node)

    Si se define, que será utilizado para localizar los viewports, si no definido, el padre de la contraseña de entrada se utilizará en su lugar. los viewports deben ser hijos de este nodo

  • viewports:

    Default: (Object)

    {
  progress: undefined,
  verdict: undefined,
  errors: undefined,
  score: undefined
}

    Un objeto que contiene los viewports a utilizar para mostrar los elementos del medidor de intensidad. Cada uno de ellos puede ser un selector CSS ( "#progressbar") o un nodo DOM de referencia. Deben ser contenidos por el container .

  • scores:

    Default: [0, 14, 26, 38, 50] (Array)

    Las puntuaciones utilizadas para determinar qué color Classes y veredictos se mostraran. debe de tener 5 elementos, lo que crea 6 categorías de la fuerza (de 6 posibles veredictos).

Ejemplo de un objeto de opciones

var options = {};
options.common = {
    minChar: 8
};
options.rules = {
    activated: {
        wordTwoCharacterClasses: true,
        wordRepetitions: true
    }
};
options.ui = {
    showErrors: true
};

Métodos

forceUpdate

Es posible forzar una actualización en un medidor de intensidad de contraseña. Se forzará un nuevo cálculo del puntaje y una actualización de los elementos de interfaz de usuario, el callback de onKeyUp se llamará .

$("#passwdfield").pwstrength("forceUpdate");

destroy

Este método eliminará los datos asociados al medidor, y los elementos de interfaz de usuario .

$("#passwdfield").pwstrength("destroy");

addRule

El plugin viene con la funcionalidad para definir fácilmente sus propias reglas personalizadas. El formato es el siguiente :

$("#passwdfield").pwstrength("addRule", "ruleName", function (options, word, score) {}, rule_score, rule_enabled);

Ejemplo:

$("#passwdfield").pwstrength("addRule", "testRule", function (options, word, score) {
    return word.match(/[a-z].[0-9]/) && score;
}, 10, true);

changeScore

Es posible cambiar la puntuación dada por una regla. Funciona de la siguiente forma:

$("#passwdfield").pwstrength("changeScore", "wordSequences", -100);

Eso penalizaría aún más la presencia de secuencias de la contraseña.

ruleActive

También es posible activar o desactivar reglas. Es tan simple como:

$("#passwdfield").pwstrength("ruleActive", "wordSequences", false);

Con ello se evitaría búsqueda de secuencias en la contraseña que se está probando .

Callback Functions

El plug-in ofrece dos funciones de devolución de llamada , onLoad y onKeyUp. Se puede usar de la siguiente forma:

$(document).ready(function () {
    var options = {};
    options.common = {
        onLoad: function () {
            $('#messages').text('Start typing password');
        },
        onKeyUp: function (evt, data) {
            $("#length-help-text").text("Current length: " + $(evt.target).val().length + " and score: " + data.score);
        }
    };
    $(':password').pwstrength(options);
});

Seguridad adicional

El plugin viene con dos reglas de validación desactivados por defecto. uno verifica demasiadas repeticiones de caracteres, y la otra regla verifica el número de clases de caracteres utilizados. Una forma sencilla de aumentar la seguridad de las contraseñas es activar estas dos reglas

$(document).ready(function () {
    var options = {};
    options.rules = {
        activated: {
            wordTwoCharacterClasses: true,
            wordRepetitions: true
        }
    };
    $(':password').pwstrength(options);
});
