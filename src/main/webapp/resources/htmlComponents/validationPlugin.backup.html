Plugin de validación jQuery


Plugin de validación jQuery

Empezar a usarlo es tan simple como usar una linea de jQuery seleccionar una forma y aplicar el plugin de validación. $("#id_de_la_forma").validate();

Además hay que agregar unas cuantas anotaciones a cada elemento HTML para indicar las reglas de validación. Por supuesto esta no es la unica forma para especificar reglas, no se tiene que confiar solo en los mensajes por default ya que estos se pueden personalizar

Para agregar el plugin a su pagina HTML solo agregue la libreria jquery.validate.js o jquery.validate.min.js en el encabezado

		
			<!-- plugin de validación -->
			<script type="text/javascript" src="jquery.validate.min.js"></script>

Metodos del plugin

Esta libreria agrega tres metodos del plugin jQuery, el punto de entrada principal va ser el metodo validate:

  • validate() - Valida la forma seleccionada.
  • valid() - Comprueba si la forma seleccionada o los elementos seleccionados son validos.
  • rules() - Lee, agrega y remueve reglas a un elemento.

Selectors personalizados

Esta libreria ademas extiende jQuery con tres selectores personalizados:

  • :blank - Selecciona todos los elementos con valor en blanco.
  • :filled - Selecciona todos los elementos con un valor capturado.
  • :unchecked - Selecciona todos los elementos que no estan comprobados.

Validator

El metodo validate retorna un objeto Validator que tiene algunos metodos publicos methods los puede usar para lanzar validación programaticamente o cambiar el contenido de la forma. El objeto validador tiene más metodos pero solamente los documentados aquí son destinados para el uso.

  • Validator.form() - Valida la forma.
  • Validator.element() - Valida un simple elemento.
  • Validator.resetForm() - Restaura la forma controlada.
  • Validator.showErrors() - Muestra los mensajes especificados.
  • Validator.numberOfInvalids() - Retorna el numero de campos no validos.

Hay algunos metodos estaticos en el objeto validator:

  • jQuery.validator.addMethod() - Agrega un metodo personalizado de validación.
  • jQuery.validator.format() - Reemplaza {n} elementos con argumentos.
  • jQuery.validator.setDefaults() - Modifica las configuraciones por default para validación.
  • jQuery.validator.addClassRules() - Add a compound class method.

Lista de metodos de validación incluidos

Un conjunto de metodos estandar de validación estan integrados:

  • required - Hace al elemento requerido.
  • remote - Solicita nu recurso para hacer la validación del elemento.
  • minlength - Hace que el elemento acepte una minima longitud.
  • maxlength - Hace que el elemento acepte una maxima longitud.
  • rangelength - Hace que el elemento acepte un rango de longitudes.
  • min - Hace que el elemento acepte un minimo.
  • max - Hace que el elemento acepte un maximo.
  • range - Hace que el elemento acepte un rango de valores.
  • email - Hace que el elemento acepte como valor un email valido.
  • url - Hace que el elemento acepte como valor una url valida.
  • date - Hace que el elemento requiera una fecha.
  • dateISO - Hace que el elemento requiera una fecha ISO.
  • number - Hace que el elemento requiera un numero decimal.
  • digits - Hace que el elemento acepte digitos solamente.
  • creditcard - Hace que el elemento requiera un numero de tarjeta de credito.
  • equalTo - Requiere que el elemento sea igual a otro.

Metodos

required() retorna: Boolean

Descripción: Hace al elemento requerido.

required()

required( expresion-dependencia )

expresion-dependencia Tipo: String

Una expresión (String) que es evaluada. Su expresión puede usar filtros de selección como: #foo:checked, #foo:filled, #foo:visible.

required( llamada-dependencia )

llamada-dependencia Tipo: Function()

La función es ejecutada y si esta retorna true entonces el elemento va ser requerido.

Ejemplos:

De un grupo de tres numeros telefonicos asegurarse que al menos uno sea capturado

		
		<form id="myform">
			<label for="mobile_phone">Telefono movil: </label>
			<input class="phone-group" id="mobile_phone" name="mobile_phone">
			<br />
			<label for="home_phone">Telefono de casa: </label>
			<input class="phone-group" id="home_phone" name="home_phone">
			<br />
			<label for="work_phone">Telefono de trabajo: </label>
			<input class="phone-group" id="work_phone" name="work_phone">
			<br />
			<input type="submit" value="Valida!">
		</form>
		
		<script>		
		$( "#myform" ).validate({
			rules: {
				mobile_phone: {
					require_from_group: [1, ".phone-group"]
				},
				home_phone: {
					require_from_group: [1, ".phone-group"]
				},
				work_phone: {
					require_from_group: [1, ".phone-group"]
				}
			}
		});
		</script>

Hace que el campo requiera una URL.

		
		<form id="myform">
			<label for="field">Capture la URL (requerido): </label>
			<input class="left" id="field" name="campoURL">
			<br/>
			<input type="submit" value="Valida!">
		</form>
		<script type="text/javascript">
		$( "#myform" ).validate({
			rules: {
				campoURL: {
				required: true,
				url: true
				}
			}
		});
		</script>

Hace al campo detalles requerido al dar click al habilitar el checkbox #otro.

		
		<form id="myform">
			<label for="otro">Dale check para hacer al otro campo requerido:</label>
			<input id="otro" type="checkbox">
			<br>
			<input id="detalle" name="detalle">
			<br>
			<input type="submit" value="Valida!">
		</form>
		
		<script type="text/javascript">
			$( "#myform" ).validate({
					rules: {
					detalle: {
					required: "#otro:checked"
					}
				}
			});
		</script>

Hace a "padre" requerido solo si edad es menor que 13.

	
		<form id="myform">
			<label>Edad: </label>
			<input id="edad" name="edad">
			<br>
			<label>Padre: </label>
			<input id="padre" name="padre">
			<br>
			<input type="submit" value="Valida!">
		</form>
		<script type="text/javascript">
		$( "#myform" ).validate({
			rules: {
				edad: {
					required: true,
					min: 3
				},
				padre: {
					required: function(element) {
						return $("#age").val() < 13;
					}
				}
			}
		});
		</script>


Usando el metodo remote

remote( options ) Retorna: Boolean

Descripción: Solicita un recurso remotamente para validar un elemento


options

Tipo: Object

Para la URL del recurso de validación de la petición hacia el servidor (String) u options para personalizar totalmente la solicitud.

Estas opciones van a profundizar extender lo prestablecido (tipoDeDato:"json", data:{nombreDelElemento:valorDelElemento}) cualquier opción que usted indique va sobreescribir a las de default.

Ejemplos:

Hace al campo email requerido, se realiza una solicitud remota a un programa que que valida si la dirección de correo capturada ya existe.

		
		$( "#myform" ).validate({
			rules: {
				email: {
					required: true,
					email: true,
					remote: "valida-email.jsp"
				}
			}
		});

Hace al campo email requerido, se realiza una solicitud remota a un programa que que valida si la dirección de correo capturada ya existe. Adicionalmente el metodo http es configurado a "post" y el username es enviado al lado de la dirección de correo.

			
			$( "#myform" ).validate({
				rules: {
					email: {
						required: true,
						email: true,						
						remote: {
							url: "valida-email.jsp",
							type: "post",
							data: {
									username: function() {
									return $( "#username" ).val();
									}
								}
						}
					}
				}
			});
